# PRD: «Пульс дня» — личный дашборд самочувствия и задач

**Версия:** 1.0
**Статус:** MVP реализован
**Тип документа:** Product Requirements Document (техническое задание)

---

## 1. Обзор продукта

«Пульс дня» — персональный веб-дашборд, который помогает человеку ежедневно фиксировать своё состояние (настроение, энергию), вести короткий список задач на день и видеть динамику за неделю/месяц. Работает как одностраничное приложение (single HTML file), данные сохраняются локально за пользователем и не пропадают между сессиями.

### 1.1 Проблема
Человеку сложно удержать в голове, как менялось его состояние за последние недели, и он не связывает самочувствие с тем, что реально было сделано за день. Готовые трекеры привычек часто перегружены функциями и не дают быстрого ежедневного ритуала «отметился за 10 секунд».

### 1.2 Решение
Единый экран, где за один визит можно:
- отметить настроение и энергию за сегодня,
- отметить/добавить задачи на день,
- увидеть стрик и тренд за неделю и месяц.

### 1.3 Целевой пользователь
Один человек, использующий личное устройство/браузер. Многопользовательский сценарий не предусмотрен.

---

## 2. Цели и метрики успеха

| Цель | Метрика |
|---|---|
| Формирование ежедневной привычки чек-ина | Длина стрика (дней подряд) |
| Прозрачность динамики состояния | Пользователь может визуально оценить тренд за 7/30 дней без дополнительных кликов |
| Низкий порог входа | Полный цикл «открыл → отметился» занимает не более 15 секунд |
| Полезность как тайм-менеджера | Пользователь фактически заводит и закрывает задачи внутри дашборда, а не в стороннем to-do |

---

## 3. Функциональные требования

### 3.1 Чек-ин настроения и энергии
- Пользователь выбирает **настроение** из 5 вариантов (эмодзи, шкала 1–5).
- Пользователь выбирает **энергию** из 5 вариантов (шкала 1–5).
- Есть необязательное текстовое поле **заметка**.
- Кнопка «Сохранить» создаёт/обновляет запись **на текущую дату**.
- Если запись за сегодня уже есть — при открытии дашборда её значения подставляются в выбор (повторное сохранение перезаписывает).
- Валидация: сохранение недоступно, пока не выбраны и настроение, и энергия.

### 3.2 Список задач на день
- Поле ввода + кнопка «Добавить» (или клавиша Enter) создают новую задачу.
- Задача отображается с чекбоксом; клик по чекбоксу или тексту переключает статус «выполнено».
- Выполненные задачи зачёркиваются, но не удаляются автоматически.
- Кнопка «×» удаляет задачу безвозвратно.
- Задачи привязаны к дате создания (текущий день); счётчик «выполнено / всего» отображается в шапке блока.
- Список задач за сегодня сохраняется отдельно от истории настроения.

### 3.3 Статистика (верхние карточки)
- **Стрик** — количество дней подряд с записью настроения, считая от сегодня назад.
- **Настроение (сред.)** — среднее по всем сохранённым записям.
- **Энергия (сред.)** — среднее по всем сохранённым записям.
- **Всего записей** — количество дней с хотя бы одной записью настроения.

### 3.4 История за 7 дней
- Список последних 7 дней (включая сегодня) с индикатором заполненности (полоса пропорциональна значению настроения) и эмодзи дня.
- Дни без записи отображаются как пустые/прочерк.
- Если записей за неделю нет вовсе — показывается пустое состояние с призывом начать.

### 3.5 Тепловая карта за 30 дней
- Сетка из 30 ячеек (по одной на день), цвет ячейки отражает среднее между настроением и энергией (градиент «холодный → тёплый»).
- При наведении — всплывающая подсказка с датой и значениями.
- Дни без данных остаются нейтрально-прозрачными.

### 3.6 Управление данными
- Кнопка «очистить все данные» удаляет всю историю настроения **и** все задачи, с подтверждающим диалогом.
- Данные сохраняются в персональном хранилище пользователя (не публичные, не доступны другим людям).

## 4. Нефункциональные требования

- **Персистентность:** данные должны сохраняться между перезагрузками страницы и повторными визитами.
- **Скорость первого отклика:** UI не должен блокироваться на время загрузки данных — показывается индикатор загрузки.
- **Отказоустойчивость:** ошибки чтения/записи хранилища не должны приводить к падению интерфейса; пользователь получает понятное сообщение об ошибке сохранения.
- **Доступность:**
  - видимый фокус на интерактивных элементах;
  - анимации фона отключаются при системной настройке «уменьшить движение»;
  - интерактивные элементы (чекбоксы, пикеры) кликабельны и доступны с клавиатуры (через Tab/Enter для полей ввода).
- **Адаптивность:** корректное отображение на мобильном экране (сетка статистики сужается до 2 колонок).
- **Автономность:** приложение — один HTML-файл без сервера и без внешних зависимостей, кроме шрифтов и API хранилища.

---

## 5. Модель данных

Хранятся два независимых объекта, ключ — дата в формате `YYYY-MM-DD`.

**Настроение/энергия** (`pulse-entries`):
```json
{
  "2026-07-16": { "mood": 4, "energy": 3, "note": "текст заметки" }
}
```

**Задачи** (`pulse-tasks`):
```json
{
  "2026-07-16": [
    { "id": "abc123", "text": "закончить макет", "done": false }
  ]
}
```

Оба объекта — приватные данные пользователя (не публичные/не общие).

## 6. Дизайн-система

- **Стиль:** премиальный glass-morphism — полупрозрачные карточки с блюром, тонкая градиентная кромка по краю, мягкие «аврора»-пятна на фоне.
- **Типографика:** Space Grotesk (заголовки/акценты), Inter (основной текст), JetBrains Mono (метки, служебные подписи).
- **Цвет:** тёмная база (глубокий тёмно-синий), акценты — холодный синий и тёплый янтарный, зелёный для состояния «выполнено/хорошо».
- **Motion:** сдержанная — плавное появление контента, лёгкий подъём элементов при наведении, медленный дрейф фоновых пятен (отключаемый).

## 7. Пользовательский сценарий (основной путь)

1. Пользователь открывает страницу → видит индикатор загрузки → данные подгружаются.
2. Видит текущую статистику и, если уже отмечался сегодня, — свои сегодняшние значения.
3. Отмечает настроение и энергию → сохраняет.
4. Добавляет 1–3 задачи на день.
5. В течение дня возвращается и отмечает задачи выполненными.
6. Периодически смотрит на недельную полосу и месячную тепловую карту, чтобы оценить тренд.

---

## 8. Вне рамок MVP (не реализуется сейчас)

- Перенос невыполненных задач на следующий день.
- Приоритеты/категории задач.
- Помодоро-таймер и учёт времени по категориям.
- Многопользовательский режим и совместные данные.
- Экспорт истории (CSV/PDF).
- Push-напоминания.

---

## 9. Дальнейшие идеи (backlog)

- Связать процент выполненных задач с общим «пульсом» дня как дополнительный фактор.
- Недельная сводка / инсайты («в дни с высокой энергией ты закрываешь больше задач»).
- Настраиваемые поля чек-ина (сон, спорт и т.д.).
